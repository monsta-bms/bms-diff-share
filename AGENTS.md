# AGENTS.md

## Project Purpose

BMS差分共有サイトを作る。

## Implementation Rules

- 既存ファイルがある場合は、新規作成ではなく上書き更新する。
- 仕様変更時は `project-docs/SPEC.md` も更新する。
- エラーは握りつぶさない。
- APIエラーは必ず JSON で `code`, `message`, `detail` を返す。
- `console.error` には処理段階名を含める。
- 初心者でも原因追跡しやすいログを残す。
- 秘密情報、APIキー、トークンをソースコードに直書きしない。
- 秘密情報は Cloudflare secrets を使う前提にする。
- 1タスクごとに変更ファイル一覧と確認手順を書く。

## Work Notes

- 指示された対象ファイル以外は変更しない。
- コードを追加・変更する前に、関連する既存ファイルと仕様を確認する。
- エラー処理を追加する場合は、呼び出し元が原因を追跡できる情報を残す。
- ログには処理段階、対象、失敗理由を含める。ただし秘密情報は出力しない。
