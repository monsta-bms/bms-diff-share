# テスト手順

## 対象

GitHub Pages の静的フロント画面で、投稿フォームUIと本番Worker API接続を確認する。

本番Worker URL:

```text
https://bms-wip-charts-worker.monsta3228gsl.workers.dev
```

GitHub Pages URL:

```text
https://monsta-bms.github.io/bms-wip-charts/
```

## 今回確認するもの

- 現行フォーム全体の2カラム配置が大きく崩れていないこと
- 投稿フォーム左カラムが `仮差分名` / `差分作者（別名義可）` / `没譜面` / `パスワードを保存` のまとまりになっていること
- 仮差分名欄が大きなtextarea風ではなく、1行入力に近い高さになっていること
- 仮差分名のplaceholderが表示されること
- 仮差分名の補足文が表示されること
- 差分作者欄が仮差分名の下に自然に詰まっていること
- 没譜面チェックが差分作者の下に表示されること
- パスワード保存チェックが没譜面チェックの下に表示されること
- 没譜面とパスワード保存の補足文が小さめに表示されること
- 左カラムの大きな余白が減っていること
- 想定難易度欄がシンボルタブ + 数字チップUIであること
- 想定難易度UIの高さ・挙動が壊れていないこと
- シンボルタブは `★` / `★★` / `sl` / `st` / `手入力` が表示されること
- シンボル切替時に想定難易度ブロックの枠の高さが変わらないこと
- 数字が常に1〜25表示されること
- PC表示で数字チップが1行10個になること
- `★` 選択時は 1〜25 が選べること
- `★★` 選択時は 1〜7 が選べ、8〜25がdisabledになること
- `sl` 選択時は 1〜12 が選べ、13〜25がdisabledになること
- `st` 選択時は 1〜15 が選べ、16〜25がdisabledになること
- `★25` から `★★` に切り替えると `★★7` に補正されること
- `★25` から `sl` に切り替えると `sl12` に補正されること
- `★25` から `st` に切り替えると `st15` に補正されること
- 手入力でも想定難易度ブロックの枠高さが変わらないこと
- 手入力タブではシンボル込みの入力ができること
- 手入力で `★12` / `sl10` / `overjoy` などが入力できること
- 手入力で数字部分が3桁以上になる入力は拒否される、または3桁目が入力できないこと
- 手入力時はシンボル別の上限チェックを行わないこと
- 選択中プレビューが `選択中: ★12` または `選択中: 未選択` のように固定表示されること
- `difficulty` と `level` が正しく送信されること
- 一覧では `difficulty` のみ表示され、`level` と重複表示されないこと
- 既存の投稿処理が壊れていないこと
- 没譜面チェック時の進捗度100固定が壊れていないこと
- パスワード保存が従来通り動作すること
- スマホ幅でも横スクロールせず表示されること
- `level` 入力欄が通常フォームに表示されないこと
- 投稿成功後に `GET /api/charts` を再取得して一覧が更新されること
- APIエラーの `code`, `message`, `detail` が画面上部に表示されること
- 送信中に投稿ボタンがdisabledになり、二重送信を防げること
- 管理パスワードをlocalStorageへ保存できること

## GitHub Pages表示確認

2. 投稿フォーム全体の2カラム配置が大きく変わっていないことを確認する。
3. 左カラムが差分識別・投稿者補助情報、右カラムが想定難易度・進捗度・管理パスワードのまとまりになっていることを確認する。
4. 想定難易度欄がシンボルタブ + 数字チップUIであることを確認する。
5. 初期表示時に上部エラー欄や各入力欄が赤エラーだらけになっていないことを確認する。

## FORM-02 左カラム余白確認

1. 仮差分名欄が大きなtextarea風ではなく、曲名やアーティスト欄に近い1行入力の高さで表示されることを確認する。
2. 仮差分名欄のplaceholderが `例: [ANOTHER] / [ALITHER] / 仮差分` であることを確認する。
3. 仮差分名欄の下に `同じ曲の別差分を区別するための名前です。` の補足文が表示されることを確認する。
4. 差分作者欄が仮差分名の下に自然な間隔で続くことを確認する。
5. 差分作者欄が通常の1行入力として表示されることを確認する。
6. 没譜面チェックが差分作者欄の下に表示されることを確認する。
7. パスワード保存チェックが没譜面チェックの下に表示されることを確認する。
8. 没譜面とパスワード保存の補足文が小さめのmuted textで表示されることを確認する。
9. 没譜面チェックとパスワード保存チェックの縦間隔が広すぎないことを確認する。
10. 左カラムに大きな空白が残らず、右カラムの想定難易度UIと視覚的にバランスが取れていることを確認する。
11. 進捗度と管理パスワードが右カラムに残っていることを確認する。
12. スマホ幅では1カラムに自然に積まれ、横スクロールしないことを確認する。

## DIF-01 想定難易度UI確認

1. 初期表示で `★` タブが選択状態、数字は未選択、プレビューが `未選択` であることを確認する。
2. `★` タブで数字が1〜25すべて表示され、disabledがないことを確認する。
3. PC幅で数字チップが `1〜10`, `11〜20`, `21〜25` の3行に分かれることを確認する。
4. `★★` タブに切り替え、1〜7が選択可能で8〜25がdisabledになることを確認する。
5. `sl` タブに切り替え、1〜12が選択可能で13〜25がdisabledになることを確認する。
6. `st` タブに切り替え、1〜15が選択可能で16〜25がdisabledになることを確認する。
7. `★25` を選択した後に `★★` へ切り替え、プレビューが `★★7` になることを確認する。
8. `★25` を選択した後に `sl` へ切り替え、プレビューが `sl12` になることを確認する。
9. `★25` を選択した後に `st` へ切り替え、プレビューが `st15` になることを確認する。
10. `★`, `★★`, `sl`, `st` を切り替えても想定難易度ブロックの外枠高さが変わらないことを確認する。
11. タブと数字チップがbutton要素として操作でき、選択中状態が見た目と `aria-pressed` で分かることを確認する。

## difficulty / level確認

`difficulty` と `level` が意図通り保存されることをAPIまたはD1で確認する。

確認例:

| 入力 | 期待するdifficulty | 期待するlevel |
| --- | --- | --- |
| `★` + `12` | `★12` | `12` |
| `★★` + `7` | `★★7` | `7` |
| `sl` + `8` | `sl8` | `8` |
| `st` + `15` | `st15` | `15` |
| 手入力 `overjoy` | `overjoy` | 空または `NULL` |

D1確認例:

```bash
cd worker
npx wrangler d1 execute wip-bms-charts-db --command "SELECT difficulty, level FROM versions WHERE title LIKE '%Difficulty Stable%' ORDER BY created_at DESC LIMIT 5;"
```

## CORSエラー時の確認

画面上で通信に失敗し、ブラウザコンソールにCORSエラーが出る場合は以下を確認する。

- `worker/wrangler.toml` に `ALLOWED_ORIGINS = "https://monsta-bms.github.io,http://localhost:8787"` があること
- `npm run deploy` 済みであること
- GitHub Pages URLではなくOrigin `https://monsta-bms.github.io` を許可していること

## テストデータ削除

必要に応じてD1とR2のテストデータを削除する。

D1は外部キーがあるため、versionから順に削除する。

```sql
DELETE FROM post_logs WHERE detail LIKE '%Initial chart version created.%' OR error_code IS NOT NULL;
DELETE FROM versions WHERE title LIKE '%Difficulty Stable%';
DELETE FROM charts WHERE chart_name IN ('[REJECTED]', '[NORMAL]', '[INVALID]', '[ANOTHER]', '[ALITHER]', '仮差分');
DELETE FROM songs WHERE title LIKE '%Difficulty Stable%';
```

R2は `charts/{chartId}/versions/root/` 配下のテストファイルをDashboardから削除する。
